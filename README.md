# Apple 통합 상담북

2026년 7월 Apple 운영 라인업 PPT를 기반으로 생성한 정적 상담 페이지입니다.

## 포함 데이터

- iPhone: 10개 제품군
- iPad: 6개 제품군
- Apple Watch: 3개 제품군
- AirPods: 5개 제품군
- Mac: 6개 제품군
- 액세서리: 41개 제품군
- AppleCare+: 47개 제품군
- 전체 모델 옵션: 620개

## 주요 기능

- 제품명, 모델코드, 색상, 용량, 호환 모델 통합 검색
- 카테고리별 탐색 및 가격 정렬
- 용량·색상·통신 방식·크기·재질 등 유효 옵션 선택
- 선택 옵션별 가격과 모델코드 확인 및 복사
- AppleCare+ 자동 연결
- 추천 비교 및 직접 비교함
- 즐겨찾기와 최근 조회 저장
- 내부 직원용 / 고객 보기 전환
- 모바일 반응형
- Apple 공식 비교 페이지 연결
- PPT 원본에서 중복되거나 형식 확인이 필요한 모델코드 경고

## GitHub Pages 배포

1. 이 폴더의 파일과 폴더를 GitHub 저장소 최상위에 업로드합니다.
2. 저장소 `Settings > Pages`로 이동합니다.
3. `Deploy from a branch`를 선택합니다.
4. `main` 브랜치와 `/ (root)`를 선택해 저장합니다.
5. 생성된 Pages 주소로 접속합니다.

별도 서버나 데이터베이스는 필요하지 않습니다.

## 로컬 사용

`index.html`을 실행하면 됩니다. 일부 브라우저의 로컬 파일 정책이 엄격할 경우 아래처럼 간단한 서버를 실행하세요.

```bash
python -m http.server 8000
```

그 후 `http://localhost:8000`으로 접속합니다.

## 데이터 갱신

`tools/build_data.py`는 PPT 표와 이미지를 다시 추출하는 빌드 스크립트입니다. 스크립트 상단의 PPT 경로를 새 파일로 변경한 뒤 실행합니다.

```bash
python tools/build_data.py
```

실행 후 `data.js`를 다시 생성하거나 제공된 단일 HTML 빌드 절차를 적용해야 합니다.

## 보안 주의

원본 운영표에는 대외비 가격과 모델코드가 포함되어 있습니다. 공개 GitHub Pages에 배포하면 누구나 소스와 데이터를 확인할 수 있으므로, 직원 전용으로 사용할 경우 접근 제한이 가능한 사내 호스팅 또는 비공개 배포 환경을 사용하세요.
